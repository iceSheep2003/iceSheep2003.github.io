浅入linux目录

linux目录天天用，但是从来没有花时间专门来了解每个目录的用途

这篇文章简要总结了linux的目录相关知识

bin

bin就是binary的简称，二进制的意思。我们经常使用的命令，均以二进制的形式保存在这个人目录下。在这个目录下，你可以找到所有的命令。为了方便理解记忆，这个目录相当于放满工具的房间。另外，这个目录下的所有文件都不需要任何权限就可以执行，bin目录是系统启动后就直接使用后的目录，也就是说，除了我们平时使用的命令行，这个目录也包括内核中使用的命令和程序。没有bin目录，系统就可能不能运行。

boot

当我们按下开机键，主板开始通电，计算机自带的BIOS检测到启动盘后，就开始读取这个目录下的内核文件，以及启动管理程序。
我们打开boot文件，排在最前面的是内核文件，

我们打开这个文件所看到的就是各种对内存，进程，驱动，以及库函数等等的详细配置。

打个比方，内核就像是小区里的物业，物业决定了小区资源的分配和管理，如果没有物业，小区就会失去秩序，操作系统也是这样。其实小到操作系统，大到与之，一个事物的创建就是一个新秩序的创建。内核之所以较内核，就是他创造了一种秩序。操作系统会按照他的秩序来运行，否则就会出现混乱。BIOS访问完内核文件后，将会从EFI中，访问系统分区，然后通过grub引导系统启动。grub定义了启动菜单的选项，超时时间。背景图片的等信息，有的系统还有grub2，他其实u就是grub的升级版。
除了内核文件，剩下的就是一些临时的镜像文件和符号表文件。有的镜像文件，就会提供一个基本的环境来处理一些可能出现的问题，比如硬件检测，设备挂载，加密锁等。

总而言之，boot就像是小区的物业一样，管理起系统的方方面面。

dev

dev其实就是device的缩写，就是设备或者装置。举个例子，当你插入一块硬盘或者多块硬盘的时候，dev目录就会以sd加abcd字母的命格式来记录这些硬盘。比如sda就是接入的第一块硬盘，a为序号，abcd这样以此类推。假如一个硬盘有多个分区的时候则以数字表示，比如sda1就是表示第二个分区。除了硬盘，所有的硬件的参数都会被保存在这个目录下。相比于windows,那么dev目录就相当于设备管理器。他们非常相似，但是对于磁盘管理方面两者有很大的区别，在linux中，没有盘符的概念

home

home目录常被称为家目录。当我们输入账号和密码的时候。按下回车键，进入的就是home目录。简单来说2,这个目录就是用于存放个人文件的目录.当然home亩路下不止存放着一个用户的一个文件
当你输入useradd -m -ikun来创建一个新用户，在和home目录下也就可以看到你刚创建的ikun用户。但是每个用户只能访问自己的home目录，root用户却可以访问所有用户的home目录。也就是说，root用户相当于是房东，其他用户相当于是租客。root可以开启和装修任何一个房间。有限的权限范围，可以让每个用户的数据相对独立，从而避免很多误操作和潜在u的风险

etc

etc目录的名字etc来源于etcetera,意思是等等。据说在早期Unix把各种零碎的文件都保存到这目录下因为没有想好一个准确的名字来命名这个目录，所以干脆就叫等等。但从他现在u的用途，不如叫做setting。为什么这么说呢，因为系统所有的设置项，都以文件的形式保存在了这个目录下。如果你刚接触linux，那一定要知道一个思想，linux的一切都是文件。比如当你要设置自己网络时你就需要打开自己的网卡配置。cd /etc/sysconfig/network-scripts/,vi ifcfg-ens33,这就是所有网卡的设置项。他其实就是一个文件。除了网卡设置，。我们还常用passwd文件，用于管理被创建的用户。shadow文件，在这里你可以查看和修改所有的用户密码。根windows一样，linux也有host文件，他也在etc目录下，用于ip映射，本地解析。crontab文件，用于定时计划任务。各个发行版和社区版的linux,etc目录下的文件都超3600多个，甚至更多，感兴趣可以自行了解。

lib

lib的全拼是library,意思是图书馆，但是在计算机世界，我们称之为库或者框架。lib目录i也是一个很底层的目录，他保存着系统，程序，甚至硬件运行所依赖的库文件。比如我们运行ls命令时，系统会调用open("/etc/ld.so.cache"),访问etc下的缓存文件，因为这个文件配置了依赖库文件的路径，然后调用open("/lib64/libselinux.so.1)访问libselinux库文件，这一步是为了检查用户的权限，确认权限后继续访问libcap文件，open("/lib64/libcap.so.2"),这个文件主要是定义所有linux系统中软件运行的POSIX标准，之后open("/lib64/libacl.so.1"),访问控制列表的相关功能。
open("/lib64/libc.so.6"),访问c的标准库，open("/lib64/libprce.so.1")这个文件是为了支持正则表达式的功能等等。lib还有很多的文件，这个例子就是想说明，不管是软件还是硬件的驱动都需要调用这个目录下已经写好的程序。可以说lib目录就是为了防止你重复造轮子，也可以让系统更加精炼简洁。

usr

usr目录很容易让人以为就是user，用户的缩写，但其实他的真身是unix software resource，软件资源目录。打开该目录，看似非常熟悉，但其实需要我们重新认识一下。

比如这里的bin目录，他保存的是我们安装软件时产生的命令，比如，zip,vim,wget等。对比根目录下的bin则是确保系统能正常操作的基本命令。那么在usr目录下的etc就是非系统应用的配置和设置。当我们使用apt-get install来安装一个python库的时候，这个python库文件就会被保存到/usr/lib/这个目录下，这个目录下的lib通常叫做非系统库。提一下libexec这个库，这个库里面的文件不是被用户执行的而是被程序调用的。其他的库，include，存放非系统程序的头文件，local，系统建议的软件安装目录，sbin,也是保存shell指令，但是只能被root用户执行，src,存放linux源代码，share,共享目录，存放帮助文档，软件说明等文件，games,游戏，tmp,用户临时缓存目录。

mnt

当我们将u盘插入电脑之后。在电脑里就会显示盘符，这是在windows,换成linux,很多人就会iui蒙圈，找不到u盘去了哪里。在linux中，你需要lsblk-p来查看,图中的sdb就是我们插入的u盘。

我们必须把他挂载到一个目录上，通常，我们会把他挂载到mnt目录上，cd /mnt,mkdir usb,mount /dev/sdb1，之后我们就可以正常访问usb。其实mnt目录并不是非常的重要，你可以挂载到其他的目录上，但是mnt是mount挂载的缩写，拥有一个较好的可读性

var

var就是Variable的缩写变量的意思，在linux中他也作为一个变量目录出现。系统和程序在运行的过程中，产生或者需要用到的文件都会被释放到这个目录下。就比如产生的日至文件，就会存放在var/log,文件下。还有/var/cache文件夹。正如其名，就是系统和程序运行产生的缓存就临时存放在这里。/var/run目录，就类似与pc电脑上的进程管理。保存了程序运行时的状态和进程编号。/var/lock则是保存记录一些正在使用的文件，将文件上锁，避免被多个程序使用。var目录是不断变化的，暂且介绍这么多。

proc

通过ps来查看系统进程，是非常常用的指令，其实这些进程信息就保存在proc,这个目录下。他的全拼是，process directory 为了内核对进程管理的访问速度和实时性，这个目录会被保存到内存中，而不是硬盘里。像proc,sys,dev,run,这样的目录，为了速度和实时性，都会被放到内存里。简单来说，proc是一个伪目录，内核为我们定义了一个管理进程的虚拟文件系统，2我们在该文件下看到的数字，其实就是以每一个进程的pid命名的目录，在这个目录里面。可以看到进程的状态进程所处在的环境变量，通过这个文件，我们可以定义这个进程使用的内存的大小，甚至可以定义你使用什么命令行来启动这个进程都可以由一个文件来定义。而所有的这些文件，在进程没有被启动的时候，都是不存在的，只有进程被启动后，彩绘被内核动态生成，并非配到内存中，关机或重启都会被清除。

sbin

注意与usr目录下的两个目录bin sbin，相区别，根目录下的bin 和 sbin,是系统启动时使用的，usr目录下的bin和sbin是运行某个程序才启动的，系统级和应用级的区别。

常用的目录暂时总结这些，常常实践才能对这些目录有更深刻的印象。
